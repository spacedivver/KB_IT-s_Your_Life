Form 관련 태그★★★★★

- 사용자로부터 text 및 각종 입력을 전달받아 page가 지정한 서버로 전달하는 태그 -> 개발자 영역
- 주요한 form : text, pw, textarea, button, summit, radio, select 등

form 기초 예제

action : 전달할 서버의 url, #은 본인의 페이지를 다시 요청하는 방법
method : 전달할 방식을 선택, get은 url을 통한 노출, post는 url과 별개로 데이터를 전달, default=get
input-id : html 레벨에서 고유의 id를 가지는 속성, 중복될수 없다!
input-name : form에서 input 파라메터 전달시 사용될 파라메터의 이름, 서버에서 활용
※ id와 name은 일치하는게 보편적이지만, 다르게 해도 문제는 없다. 각자 식별만 잘하면 된다.

form 복합 예제

value : input값의 초기값을 지정하는 속성, 서버단에서 주로 입력이 이뤄짐
readonly : input 필드를 읽기 전용으로 만드는 속성, 수정 불가
required : submit시의 필수 데이터를 체크하는 속성
minlength/maxlength : 최소/최대 입력값 제한거는 속성
placeholder : 입력에 대한 예시 또는 힌트를 보여주는 속성
