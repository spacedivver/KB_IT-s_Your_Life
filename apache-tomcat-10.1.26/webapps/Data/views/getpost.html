Get 방식과 Post 방식 차이점 실습하기

1. Get 방식 전송

- URL을 통해 파라메터를 전달하는 방법 ex) test.com?name1=value1&name2=value2 ....
- 활용 : 일반적인 페이징 탐색이나 대부분의 요청은 get 방식으로 처리 된다.
- 장점 : 히스토리가 남아 페이지 탐색에 적합하고, 간결하고 내부적으로 처리가 빠르다.
- 단점 : 네트워크 경로에 url 노출됨으로 보안에 취약하고 길이 제한이 있다. (2kb)

2. Post 방식 전송

- URL과 요청 데이터를 분리(URL, Body)하여 전송하는 방법
- 활용 : 사용자의 입력 데이터를 처리할때 주로 사용한다. (로그인, 회원가입, 게시판 글쓰기 ...)
- 장점 : 데이터 크기 제한이 없다. 보안적으로 get 방식보다 안전하다. *(https까지 붙어야 안전하다!)
- 단점 : 처리가 복잡하다.(서블릿에서만, 스프링에서는 차이 없다.) 히스토리가 안남아 탐색에는 부적합하다.
